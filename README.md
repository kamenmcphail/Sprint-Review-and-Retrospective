# Sprint-Review-and-Retrospective
CS250 Final Project

To implement user needs into a program, user research needs to be conducted to define user profiles, feedback needs to be gathered and analyzed, and those needs need to be priortized. Creating user stories helps the development team to prioritize user needs from the user's persepctive, since that is how the user stories are written. This ensures that the development team remains focused on delivering a valuable product to end users. I hope to incorporate prioritzing communication and collaboration among team members into my future work, along with adapting to changes and focusing on continuous improvement. These are all qualities of what it means to be a good team member in software development.
